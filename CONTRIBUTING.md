# Contributing

Thank you for contributing to awesome-sre-agents.

## Adding an entry

Before opening a pull request, check that the item meets these criteria:

- It is an AI agent or agent-adjacent tool with a meaningful SRE or DevOps use case.
- It has a public URL (product page, GitHub repo, or documentation).
- The project is actively maintained. Abandoned or unmaintained projects will not be added.
- The description in your PR is one factual sentence that states what the tool does, not marketing copy.

## Format

Each entry follows this pattern:

```
- [Name](URL) - One sentence describing what it does.
```

For open-source projects, add `(Open Source)` before the dash:

```
- [Name](URL) (Open Source) - One sentence describing what it does.
```

Keep entries sorted alphabetically within each section.

## Pull request checklist

- [ ] Entry is in the correct section.
- [ ] Entry is sorted alphabetically within the section.
- [ ] Description is one factual sentence with no marketing language.
- [ ] URL resolves and loads correctly.
- [ ] No duplicate of an existing entry.

## Updating an existing entry

If a URL has changed or a description is outdated, open a PR with the corrected information and a brief note on what changed.

## Removing an entry

If a project has shut down or gone unmaintained, open an issue or PR with a link to evidence (for example, a shutdown announcement or a repo archived more than 12 months ago).
